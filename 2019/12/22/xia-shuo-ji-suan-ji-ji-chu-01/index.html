主体内容

什么是CPU

CPU长什么样

CPU有什么用

    CPU ：中央处理器 （Central Processing Unit）是计算机的主要设备之一，功能主要是解释计算机指令以及处理计算机软件中的数据（以上摘自 wiki 百科：https://zh.wikipedia.org/wiki/中央处理器 ）。

    也就是说，所谓CPU，就是计算机的大脑，用于处理指令和数据。
    举个例子：请对下面的式子进行计算，并给出计算结果：
                * 1 + 1 = 
    我们根据题干可以得出，我们要做的，就是计算“1 + 1 = ”，那么，我们根据题干去做计算，就是根据指令去做相应的动作，而“1 + 1 = ”就是数据，当我们得出“1 + 1 = 2”的时候，我们就是对数据进行了处理。
    而不论我们对指令做出反应，还是得出“1 + 1 = 2”，都是通过大脑来完成，或者说，通过大脑来控制身体其它部分来完成。
    而这，就是CPU的作用，即：对指令做出反应以及对数据进行处理。

CPU的优缺点：

  • 优点：速度快
  • 缺点：容量小

什么是内存

内存长什么样

内存有什么用

    内存：随机存取存储器（Random Access Memory，缩写：RAM；也叫主存）是与CPU直接交换数据的内部存储器。可以随时读写（刷新时除外），而且速度很快，通常作为操作系统或其他正在运行中的程序的临时数据存储介质。
    主存（Main memory）即电脑内部最主要的存储器，用来加载各式各样的程序与数据以供CPU直接运行与运用。（以上摘自 wiki 百科：https://zh.wikipedia.org/wiki/随机存取存储器 ）。

    所谓内存，就像快递点，可以帮你临时存放你的快递，但不能永久帮你保存。它可以与CPU直接通信（也就是这两个可以直接勾搭上）。
    为什么会出现它呢？
    因为CPU虽然速度快，但奈何容量小，若数据太多，那么CPU就会无处将它们安放，也就会导致一部分数据丢失，这明显不是我们所想要的，于是，就出现内存。
    CPU会将要处理的数据读取到CPU中进行处理，而待处理的数据就存放在内存中。
    实际上，在CPU和内存间还有个叫 cache 的家伙，但它不是今天的重点，先略过。

内存的优缺点

  • 优点：速度快（相对于硬盘）、容量大（相对于CPU），可与CPU直接通信
  • 缺点：断电清空，即断电之后，所有数据都将丢失

什么是硬盘

硬盘长什么样

硬盘有什么用

    硬盘（Hard Disk Drive，缩写：HDD）是电脑上使用坚硬的旋转盘片为基础的非易失性存储器，它在平整的磁性表面存储和检索数字数据，数据通过离磁性表面很近的磁头由电磁流来改变极性的方式被写入到磁盘上，数据可以通过盘片被读取，原理是磁头经过盘片的上方时盘片本身的磁场导致读取线圈中电气信号改变。硬盘的读写是采用半随机存取的方式，可以以任意顺序读取硬盘中的数据，但读取不同位置的资料速度不相同。硬盘包括一至数片高速转动的盘片以及放在执行器悬臂上的磁头。（以上摘自 wiki 百科：https://zh.wikipedia.org/wiki/硬盘 ）。

    所谓硬盘，就像你的家，可以永久保存你的东西，前提是这些东西可以永久不损坏，或者你家永久不变。它不能和CPU直接通信（也就是不能直接勾搭上）。
    为什么会出现它呢？
    因为有些操作又是需要重复的，比如播放音乐，而内存有具有断电清空的特点。如果每次要播放音乐都需要写一次播放音乐用的程序（程序可以简单理解成一些指令的集合，就像一张试卷一样）。就像隔壁村的王木匠每次要用到锤子的的时候，都要重新造一个一样，浪费了不必要的时间。
    因此，人们便想着将那些会经常用到的程序存放起来，需要的时候就将其读取到内存中，再由CPU进行处理，于是，硬盘出现了。
    从此，每次听音乐不再需要现写一个程序，隔壁村的王木匠也有了一个长期可用的锤子，不需要在每次要用的时候重新造了。

硬盘的优缺点

  • 优点：容量大，可长期存储数据且断电后数据不丢失
  • 缺点：速度慢，无法与CPU直接通信

它们的关系

案例

    先来个案例：
    有个小伙子叫阿麦，他要去考科目一，他去到了车管所，发现要考场的布局就像下面那张图一样

    阿麦首先要在休息区存放随身携带的物品，因为这时候考试区里面有人在考试，等待区又有满员了，于是他便在休息区等待考官的安排以进入待考区等待考试。[1]
    过了一会，他发现等待区已经没人了，但是考官依然没让他进入待考区，他按捺不住，上前询问考官。
    考官：“你是几点的考场？”
    阿麦：“我是早上10点的。”
    考官：“现在是早上8点的考场，你先回去休息区坐着等吧。”
    阿麦看了看时间，现在才9点30，还没到10点，没办法，他只能回到休息区继续等待。
    过了一会，时间到了，考官过来了，让休息区中，早上10点考场的人现在排队，依次进入待考区，因为阿麦排的比较靠后，所以进了待考区后，他依然需要等待。[2]
    当代考区满后，考官把门关上，避免其他人再进入待考区，这时候，考官说了考试记录和注意事项之后，让待考区的人依次到前面刷身份证，等待考试座位的安排，拿到考试座位后即刻进入考场进行考试。
    因为阿麦排在后面，所以需要等待前面的人进考场后才能去获取考试座位安排，虽然心里有些急，但还是得等着。[3]
    眼看就要到他了，突然考官带进来一位孕妇，考官让她先进去考试，当她进入考场之后，考场人数满了，阿麦没办法只能继续等。[4]
    又过了一会，有人考完走了，于是阿麦开开心心地刷身份证，拿了考试座位安排就进入考场考试去了。
    考完之后，他又回到代考区，然后从待考区回到了休息区，取了个人物品之后，回学校去了。[5]
-------------------------------------------------------------------
    分析：
    [1] 阿麦要进入考场考试必须要先进入待考区
    [2] 阿麦必须满足条件才能进入待考区
    [3] 即使进了待考区，阿麦仍需排队
    [4] 即使轮到阿麦进入考场，但遇到特殊情况，阿麦仍需往后排
    [5] 阿麦考完试后要回到休息区，必须先回到待考区

三者的关系

1、CPU就是案例中的考场

2、内存就是案例中的待考区

3、硬盘就是案例中的休息区

4、硬盘的数据要进入CPU，由CPU处理，就必须先进入内存（对应案例分析的[1]）

5、即使内存没有满，但硬盘的数据依然不能随意进入内存，需要满足一定条件才能进入内存，等待CPU的处理（对应案例分析的[2]）

6、即使硬盘数据进入了内存，仍然需要排队（对应案例分析的[3]）

7、即使轮到该数据进入CPU，由CPU处理，但如果有特殊情况（如系统安全等优先级较高的事件），该数据依然要往后排（对应案例分析的[4]）

8、数据处理完后，若要回到硬盘，依然要经由内存，而不能直接回到硬盘

9、由硬盘 到 内存 再到 CPU 的过程叫做读取

9、由CPU 到 内存 再到 硬盘 的过程叫做写入

总结

  • CPU就像军队中的指挥部，内存就像是通讯兵，硬盘就像营区。指挥部的指令不能直接下到基层士兵，基层士兵也不能直接向指挥部汇报，所以要有通讯兵来充当二者的桥梁
  • 计算机中，这三者是最常见的，比如买电脑的时候，所以我才写这篇文章，其它的关于计算机组成的知识，推荐去看书
  • 这篇文章在我想写的 二级系列 和 C 语言系列 中，都是第一篇，因为二级中涉及到的栈和队列等计算机基础知识，和 C 语言里面，指针、数组的知识，都需要对内存有一定了解，而内存又和CPU和硬盘无法分离，所以我就放一起了
